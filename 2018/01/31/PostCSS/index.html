postCSS结合Grunt

postCSS

postCSS到底是个什么鬼？？？

git

官网

总结一下：

它提供了一种用 JavaScript 代码来处理 CSS的方法.

它负责把 CSS 代码解析成抽象语法树结构（Abstract Syntax Tree，AST），再交由插件来进行处理.

插件基于 CSS 代码的 AST 所能进行的操作是多种多样的，比如可以支持变量和混入（mixin），增加浏览器相关的声明前缀，或是把使用将来的 CSS 规范的样式规则转译（transpile）成当前的 CSS 规范支持的格式

PostCSS 的强大之处在于其不断发展的插件体系。

目前 PostCSS 已经有 200 多个功能各异的插件。开发人员也可以根据项目的需要，开发出自己的 PostCSS 插件。

主要功能

第一个就是前面提到的把 CSS 解析成 JavaScript 可以操作的 AST

第二个就是调用插件来处理 AST 并得到结果

因此，不能简单的把 PostCSS 归类成 CSS 预处理或后处理工具。

使用

PostCSS 一般不单独使用，而是与已有的构建工具进行集成
